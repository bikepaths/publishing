## EPUB 3.3 Specification Requirements

The current production standard is EPUB 3.3, maintained by the W3C. Every epub produced on this platform must conform to this specification and pass EPUBCheck validation before submission to any distribution channel.

### Required Package Components

| Component | File | Purpose |
|---|---|---|
| Package Document | `content.opf` | Manifest, spine, metadata — generated by pandoc |
| Navigation Document | `nav.xhtml` | TOC, landmarks, page-list — generated by pandoc |
| NCX (legacy) | `toc.ncx` | Backward compatibility with EPUB 2 readers — generated by pandoc |
| Content Documents | `ch001.xhtml` etc. | One per chapter/section — generated by pandoc |
| Stylesheet | `stylesheet.css` | Typography and layout |
| Cover Image | `cover.jpg` | Embedded optimized JPEG (under 1MB) |

Pandoc generates all required components automatically when invoked with `--to epub3`. Manual editing of the package internals is not required and not recommended.

### `epub:type` Semantic Inflection

EPUB 3 uses `epub:type` attributes on HTML elements to communicate document structure to reading systems. Pandoc applies these automatically. The CSS template targets them directly. Key values:

| Value | Applied To | Meaning |
|---|---|---|
| `cover` | First section | Cover image page |
| `titlepage` | Front matter section | Book title and author |
| `copyright-page` | Copyright section | Rights and publication info |
| `frontmatter` | Pre-chapter sections | Author's note, preface |
| `chapter` | Each chapter section | Body content |
| `bibliography` | Citations section | Reference list |
| `toc` | Navigation document | Table of contents |

### Accessibility Requirements (EPUB Accessibility 1.1)

KDP and the broader distribution ecosystem increasingly require epub accessibility compliance. The following are non-negotiable for commercial distribution:

- All images must carry meaningful `alt` text. Cover images: `alt="Cover of [Title] by [Author]"`.
- The navigation document (`nav.xhtml`) must include a landmarks nav element.
- The document language must be declared in the package metadata (`xml:lang`, `lang`).
- Reading order must match the logical order of content in the spine.
- No content may be conveyed by color alone.

## CSS Architecture Standards

### Typography Hierarchy

| Element | Font | Size | Weight | Notes |
|---|---|---|---|---|
| Body | System Default | 1em | normal | Justified, hyphens enabled |
| H1 (Chapter title) | System Default | 1.6em | bold | Page break before, left aligned |
| H2 (Location line) | System Default | 0.95em | normal | Italic, subdued color |
| H3 (Sub-section) | System Default | 0.9em | bold | Within-chapter use only |
| Paragraph | System Default | 1em | normal | No text-indent, 1em bottom margin |
| Citations | System Default | 0.88em | normal | Hanging indent -1.8em / 1.8em |
| Cover image | Yes | Optimized JPG (under 1MB) for EPUB; original PNG/JPG (min 2500px) in handoff |

### Page Break Rules

Chapter H1 headings must trigger a page break before. Use both the legacy and current properties for maximum reader compatibility:

```css
h1 {
    break-before: page;
    -webkit-break-before: page;
    page-break-before: always;
}
```

Suppress page breaks on the first H1 of the cover and title page sections using `epub:type` selectors.

### Widows and Orphans

Set on the body element:

```css
body {
    widows: 2;
    orphans: 2;
}
```

This prevents single lines from being stranded at the top or bottom of a screen/page. Not all reading systems honor this, but it is correct practice and causes no harm when ignored.

## Citation Standard: Chicago 17th Edition Bibliography

All citations published within the platform use Chicago 17th edition Bibliography format. This is the standard for humanities, social science, and policy non-fiction. Do not use APA, MLA, or journalism style within book-length works.

### Journal Article

```
Last, First, First Last, and First Last. "Article Title." *Journal Name* Volume, no. Issue (Year): Pages. https://doi.org/xxx.
```

### Book

```
Last, First. *Book Title: Subtitle*. Place: Publisher, Year.
```

### Government / Institutional Report

```
Agency Name. *Report Title*. Place: Publisher, Year. URL.
```

### Website / Blog Post

```
Last, First. "Post Title." *Publication Name* (blog). Month Day, Year. URL.
```

### Working Paper / Preprint

```
Last, First. *Paper Title*. Series Name Working Paper [Number]. Month Year. URL.
```

### Dissertation

```
Last, First. "Dissertation Title." PhD diss., University Name, Year.
```

### Rules for Citation Entries in ePub

1. Every entry must be verifiable by a reader with access to the internet.
2. DOIs must be formatted as full URLs: `https://doi.org/xxx`, not bare DOI strings.
3. No placeholder authors. If an author cannot be confirmed, remove the entry.
4. All entries must be wrapped in the pandoc fenced div: `::: {.citations}` / `:::`.
5. Chapter subheadings within the bibliography use H2, which the CSS styles with a bottom border for visual separation.

## EPUBCheck Validation Mandate

EPUBCheck is the official conformance checker for the EPUB specification, maintained by the W3C/DAISY Consortium. Version 5.x validates against EPUB 3.3.

### Error Severity Levels

| Level | Action Required |
|---|---|
| ERROR | Must fix before submission. The epub does not conform to the specification. |
| WARNING | Review and resolve where possible. The epub is technically valid but may behave unexpectedly in some reading systems. |
| INFO | No action required. |

### Common EPUBCheck Errors and Resolutions

| Error Code | Cause | Resolution |
|---|---|---|
| `CSS-008` | CSS property not supported in EPUB | Add fallback property or remove unsupported value |
| `OPF-086` | Missing required metadata | Add `dc:language`, `dc:title`, or `dc:creator` to package metadata |
| `RSC-007` | Resource referenced but not in manifest | Ensure all linked files are included in the epub package |
| `NAV-003` | Navigation document missing landmarks | Pandoc generates this automatically; check pandoc version |

## KDP Technical Specifications

### eBook (Kindle)

| Parameter | Requirement |
|---|---|
| Format | EPUB 3 preferred; MOBI/KF8 accepted but deprecated |
| Cover image | Minimum 2,500px on longest side; JPG or PNG; sRGB; under 50MB (Dashboard upload); under 1MB (Embedded EPUB JPG) |
| Font embedding | Optional; platform fonts render on all Kindle devices |
| DRM | Optional at upload; set at title level |
| Pricing for 70% royalty | $2.99–$9.99 list price |

### Print (Paperback)

| Parameter | Requirement |
|---|---|
| Format | PDF (recommended) or DOCX |
| Interior color | Black and white standard; color adds significant per-unit cost |
| Bleed | 0.125 inch on all sides for full-bleed covers |
| Cover | Separate file; KDP Cover Creator or print-ready PDF |
| Spine width | Calculated by KDP based on page count and paper type |
